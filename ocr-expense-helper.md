# 開発依頼：領収書・請求書の簡単仕訳・経費精算ヘルパー（FastAPI + SQLite + OCR）

アップロードされた領収書や請求書の画像（PNG/JPEG）からOCR技術で文字を自動抽出し、「日付」「金額」「支払先（発行元）」を自動で判定・データベースに一時保存し、最終的にCSVとして一括エクスポートできるWebアプリケーションを開発してください。

## 1. システム構成・ファイル構成
このプロジェクトは、バックエンド（Python）とフロントエンド（HTML/CSS/JS）を分離した以下の構成にします。

- `/app/main.py`（FastAPI：APIサーバー、OCRロジック、SQLiteデータベース操作、CSV生成）
- `/app/database.py`（SQLiteの初期化、テーブル定義、CRUD操作用関数）
- `/frontend/index.html`（Tailwind CSSを使用したモダンなアップロード＆データ管理画面）
- `/frontend/style.css`（ドラッグ＆ドロップエリアや一覧テーブル用などのカスタムスタイル）
- `/frontend/script.js`（非同期通信（Fetch API）によるファイル送信、APIからのデータ取得・描画処理）

## 2. 実装要件

### 【バックエンド（Python / FastAPI / SQLite / Tesseract）】
- **データベース定義（SQLite / database.py）：**
  - テーブル名: `expenses`
  - カラム: `id` (PK, AutoIncrement), `date` (日付), `amount` (金額), `vendor` (支払先・発行元), `raw_text` (OCR生のテキストデータ), `created_at` (登録日時)
- **APIエンドポイント（main.py）：**
  - `POST /api/upload`：
    - 画像（JPEG/PNG）を受け取る。
    - **OCR処理：** `pytesseract` を使用して画像から日本語テキストを読み取る。
    - **自動抽出（パーサー）：** 正規表現等を用いて、テキストから「日付（例：2026年7月17日、2026/07/17など）」「金額（例：￥5,000、5,000円、計 5000 など）」「支払先（テキスト冒頭や株式会社などの名称）」を自動で推測・抽出する。
    - 抽出したデータをSQLiteデータベースに新規レコードとして保存する。
    - 抽出・保存に成功したデータをJSONで返却する。
  - `GET /api/expenses`：保存されている仕訳データ一覧をJSONで取得する。
  - `PUT /api/expenses/{id}`：自動抽出に誤りがあった際、ユーザーが手動でデータを修正できるように更新処理を行う。
  - `DELETE /api/expenses/{id}`：データを削除する。
  - `GET /api/export`：保存されている全データを「弥生会計/マネーフォワード等のインポート形式に近い汎用CSV形式」で生成し、ファイルとしてダウンロード（エクスポート）させる。

### 【フロントエンド（HTML/CSS/JS）】
- **直感的なファイルアップロード：**
  - ドラッグ＆ドロップ、またはファイル選択に対応したお洒落なアップロードエリア（アップロード中はスピナー等でローディング状態を表現）。
- **確認・編集インタフェース：**
  - アップロード完了後、OCRで自動抽出された「日付」「金額」「支払先」をその場でフォーム内に表示し、万が一誤認識があってもユーザーが直接手動で修正・保存できるUI。
- **保存データ一覧（データベース連携）：**
  - 画面下部に、これまでに蓄積された経費データの一覧をスッキリとしたテーブル形式で表示。
  - テーブル内には「手動編集」および「削除」ボタンを設置。
- **エクスポート：**
  - 「CSVダウンロード」ボタンを設置。1クリックでバックエンドの `/api/export` を叩き、ファイルをダウンロードさせる。

## 3. 注意点・前提条件
- Pythonの外部ライブラリは `fastapi`, `uvicorn`, `pytesseract`, `pillow` などを想定した構成にしてください。
- 外部APIキーなどは必要とせず、ローカル（サーバー内）で動作する `pytesseract`（Tesseract OCR）を利用します。
- セキュリティのため、ファイルのサイズ制限（最大10MB）などのバリデーションをバックエンド側で設けてください。
